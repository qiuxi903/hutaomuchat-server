## 第一轮：基础架构与核心功能 (2026-06 初)

### 项目搭建
- 使用 Node.js + Express + ws 搭建后端服务
- 实现 JWT 认证体系（注册/登录/token 刷新）
- 内存数据库存储所有数据

### 核心 API
- **认证**：注册（手机号+密码+昵称）、登录、获取/更新个人资料
- **用户**：搜索用户、获取用户详情
- **好友**：发送/接受/拒绝好友请求、好友列表
- **聊天**：创建单聊/群聊、聊天列表、聊天详情
- **消息**：消息历史
- **朋友圈**：发布动态、点赞、评论
- **文件上传**：multer 处理头像/图片/表情包，7 天自动清理

### WebSocket
- ws 库实现实时通信
- 消息转发，不存储

## Round 24 (2026-06-19)

### 个推 UniPush 2.0 集成

#### 架构
```
发送方 → 服务端 → [在线] WebSocket 实时推送
                  → [离线] 个推 UniPush → 厂商推送通道 → 设备系统服务 → 通知栏
```

#### 服务端变更
1. **getui.js（新增）** — 个推 REST API v2 模块
   - SHA-256 鉴权获取 auth token（24小时有效）
   - `sendPushToSingle(cid, title, body, payload)` 向单个用户发送推送
   - 从 `getui.config.json` 读取 appId/appKey/masterSecret
   - 无配置文件时自动跳过

2. **db.js** — 新增 `pushTokens` 集合
   - `setPushToken(userId, cid)` / `getPushToken(userId)` / `removePushToken(userId)`

3. **routes/push.js（新增）** — 推送 token 管理 API
   - `POST /api/push/token` — 注册 CID
   - `DELETE /api/push/token` — 移除 token

4. **ws/index.js** — 离线消息 fallback 到推送通道
   - 用户不在线时通过 UniPush 发送系统级推送

5. **server.js** — 注册 push 路由

- 英文：Server Settings、Edit Settings、GeTui Configuration、Danger Zone 等

---

## 技术架构

**服务端**：Node.js + Express + ws，JWT 认证，内存数据库 + JSON 文件持久化（Proxy 自动拦截 + 防抖写入 + 原子写入），multer 文件上传（10MB 限制），7 天自动清理。个推 UniPush 2.0 离线推送。管理后台 SPA（暗/亮主题，中英文，自动刷新）。OOBE 首次部署向导。

**通信协议**：REST API 处理元数据（用户/好友/聊天/群设置/表情包/入群申请），WebSocket 处理实时消息（文本/图片/文件/视频/typing/入群通知/撤回/@全体）。
